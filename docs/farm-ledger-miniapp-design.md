# 农事记账小程序界面设计路线

## 设计目标

保留当前小程序的深色底、浅蓝描边、高对比文字和卡片式信息层级，但减少表格式输入感。界面应像一个「农场经营助手」，让用户围绕每天发生的事来记录账目和农事，而不是在多个格子里找字段。

## 设计原则

1. 首页只回答三个问题：今天做了什么、本月花了多少、还有什么需要补记。
2. 记账和农事不拆成两个孤立系统，一次农事可以顺手关联成本。
3. 高频入口大而少，低频字段后置，避免首页堆满分类按钮。
4. 列表以时间线呈现，弱化表格感，强化「日志」感。
5. 报表只给经营结论，不做后台管理式数据面板。

## 视觉方向

### 延续现有风格

- 背景：继续使用深色底，建议为近黑灰而不是纯黑。
- 卡片：保留圆角、浅蓝描边、顶部浅蓝高亮边。
- 主文字：白色或近白色，保证户外使用可读性。
- 辅助文字：灰蓝色，减少信息噪音。
- 强调色：沿用浅蓝作为选中、按钮和重点边框。

### 优化建议

- 金额不要全部用亮蓝，主金额可以用白色，增长、完成、收入等正向状态再用绿色。
- 农事标签使用低饱和色块，例如浅蓝、浅绿、浅黄，避免一屏都是描边按钮。
- 卡片内部减少网格线，改用分组间距和小标题建立层级。
- 重要操作按钮使用整块填充色，次要筛选使用轻描边。

## 推荐方案：经营首页 + 事件时间线

这是比现有 A/B/C 更适合的方案。它保留小程序的暗色科技感，但把「填格子」改成「记录农场每天发生的事」。

## 信息架构

底部导航建议保留 3 个 Tab：

1. 首页
   - 今日概览
   - 快速记录
   - 最近动态
2. 记录
   - 时间线
   - 筛选
   - 详情
3. 报表
   - 本月投入
   - 成本构成
   - 地块对比

不建议把「成本」和「农事」做成两个一级 Tab，因为真实使用中它们经常发生在同一件事里。

## 首页设计

### 顶部区域

结构：

- 日期：今天 / 7月2日
- 地块切换：全部、东片、西片
- 轻量天气或备注位：可选，不强制

示例：

- 今天 · 全部地块
- 本月已记录 12 笔，农事 5 次

### 核心概览卡

只突出一个主数字：

- 本月投入：¥7,660

辅助信息：

- 今日支出：¥860
- 今日农事：打药 1 次
- 最高成本：肥料 ¥3,200

右上角放一个小入口：

- 看报表

### 快速记录区

不要展示 6 个以上分类格子。建议两个主入口：

1. 记一笔花费
   - 说明：肥料、农药、用工、农机
2. 记一件农事
   - 说明：打药、浇水、施肥、巡田

两个入口使用大卡片样式。主卡片可浅蓝填充，次卡片可深色描边。

### 最近动态

用时间线替代账单表格。

示例：

- 上午 9:30
  - 东片 · 打药
  - 农药 ¥860
- 昨天
  - 西片 · 施肥
  - 肥料 ¥3,200
- 6月30日
  - 全社 · 用工
  - 人工 ¥1,200

每条动态右侧可用小标签区分：

- 成本
- 农事
- 成本 + 农事

## 记账流程

目标是两步完成，第三步只做可选补充。

### 第一步：选费用类型

展示 6 个大类：

- 肥料
- 农药
- 用工
- 农机
- 种苗
- 其他

分类不放首页，进入「记一笔花费」后再展示。

### 第二步：填金额

核心输入：

- 金额

辅助输入：

- 地块
- 日期
- 备注

金额键盘应成为主视觉，减少用户找输入框的负担。

### 第三步：关联农事

保存前给轻量选项：

- 是否关联一件农事？
- 例如：这笔农药是否对应「打药」？

用户可跳过，不强制。

## 农事日志流程

### 第一步：选农事类型

- 打药
- 施肥
- 浇水
- 除草
- 采收
- 巡田

### 第二步：选地块

- 东片
- 西片
- 全部
- 其他

### 第三步：补充信息

根据农事类型动态显示字段：

- 打药：药名、用量、次数
- 施肥：肥料、用量、方式
- 浇水：时长、方式
- 除草：方式、面积

底部提供：

- 顺便记成本

这样农事日志不会变成固定长表单。

## 记录页设计

记录页使用时间线和筛选条。

### 顶部筛选

第一行：

- 本月
- 全部地块
- 全部类型

第二行可横向滚动：

- 成本
- 农事
- 打药
- 施肥
- 用工

### 内容结构

按日期分组：

- 7月2日 今天
  - 东片 · 打药 · ¥860
  - 西片 · 浇水
- 7月1日
  - 西片 · 施肥 · ¥3,200

有金额的记录显示金额，没有金额的农事显示农事摘要。

## 报表页设计

报表页不要做成复杂仪表盘，只保留三张经营卡。

### 卡片一：本月投入

- 主数字：¥7,660
- 说明：比上月多 / 少多少，可后续添加
- 小条形分布：肥料、农药、用工、农机

### 卡片二：成本构成

列表形式比饼图更适合小程序：

- 肥料 ¥3,200 42%
- 农药 ¥860 11%
- 用工 ¥1,200 16%

每行使用细进度条。

### 卡片三：地块对比

- 东片 ¥4,200
- 西片 ¥2,800
- 全社 ¥660

地块对比适合做横向条形图，不建议用表格。

底部动作：

- 导出 CSV
- 分享长图

## 页面样式细节

### 卡片

- 圆角：16 到 20
- 边框：1px 浅蓝透明度 60%
- 顶部高亮边：可保留，但不要每张卡都过强
- 内边距：16 到 20

### 按钮

- 主按钮：浅蓝填充，深色文字
- 次按钮：深色底，浅蓝描边
- 标签按钮：小尺寸、低对比，避免抢主按钮

### 文案

避免：

- 记一笔：6 宫格点类型 -> 只填金额 -> 完成
- 顶部筛选：月份 | 类型 | 地块

改为：

- 今天花了多少
- 今天做了哪些农事
- 给东片补记一笔
- 顺便关联成本

文案要像给种植户看的工具，不像后台系统。

```text

本月已记录 12 笔，农事 5 次

[ 本月投入卡 ]
¥7,660

肥料 ¥3,200       用工 ¥1,200

[ 记一笔花费 ]
肥料、农药、用工、农机

[ 记一件农事 ]

昨天       西片 · 施肥        肥料 ¥3,200
6月30日    全社 · 用工        ¥1,200
```

## 推荐记录详情样式

详情页不要像表单回显，而像一张事件卡。

示例：

```text
东片 · 打药
7月2日 上午 9:30

成本
农药 ¥860

补充
药剂：除草剂
次数：1 次
备注：雨后补打

[编辑] [删除]
```

## 最终取舍

建议舍弃现有三个方案的主结构，但保留现有视觉风格。

- 不用方案 A 的首页宫格，因为仍然像分类面板。
- 不用方案 B 的成本 / 农事双 Tab，因为会把同一件事拆开。
- 不用方案 C 的三步入口，因为每次都像被引导填流程。

最终采用：

- 首页：经营概览
- 记录：事件时间线
- 新增：记花费 / 记农事两个入口
- 报表：三张经营结论卡

一句话：界面继续保持你的小程序深色卡片风格，但交互从「填格子」变成「记今天发生的农场经营事件」。
